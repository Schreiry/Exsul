## 1. Общее описание проекта

**EXSUL** — это закрытая peer-to-peer система учёта и отчётности производства для малого семейного бизнеса.  
Программа **не имеет центральных серверов**. Каждое устройство (компьютер), добавленное в сеть узлов, является равноправным узлом (node). Данные синхронизируются напрямую между устройствами.

- **Цель:** Полностью заменить Excel/Google Таблицы для семейного бизнеса по выращиванию и продаже цветов (и в будущем — других направлений).
- **Целевая аудитория:** люди без технического образования. Интерфейс должен быть максимально интуитивным, «как в мобильном приложении».
- **Текущий стек:**
  - Frontend: **SvelteKit 5** (Svelte Runes)
  - Backend: **Tauri v2 + Rust**
  - База данных: **SQLite** (rusqlite) + миграции
  - Синхронизация: CRDT + event-sourcing с Hybrid Logical Clock (HLC)

Ссылка на репозиторий: https://github.com/Schreiry/Exsul

## 2. Ключевая концепция — «Режимы бизнеса» (Business Modes / Presets)

Это **самая важная архитектурная особенность** проекта.

- Сейчас существует **три предустановки** (в будущем их будет больше).
- Каждая предустановка — это **полностью изолированный слой** программы.
- У каждой предустановки:
  - Своя база данных (файлы БД разные).
  - Свои настройки (`app_settings`).
  - Свои расчёты, логику, тексты, компоновку экранов.
- **Визуальный стиль и общая навигация** — единые для всех режимов (тема, glass-morphism, цвета, шрифты).
- Переключение режима происходит через `$preset` (в коде используется проверка `$preset === 'flowers'` и аналогичные для других режимов).

**Режим «flowers» (Цветы)** — основной и единственный активно разрабатываемый на текущий момент.  
Все слова в промптах, связанные с:
- цветы, цветок, сорт, стебли, букет, упаковка, упаковки, теплица, оранжерея, склад, harvest, packaging, stems_per_pack, greenhouse и т.д.

— **автоматически относятся только к режиму «flowers»**. Никогда не распространяй логику цветов на другие режимы без явного указания.

Другие режимы защищены. Изменения в них должны быть изолированы.

## 3. Архитектура P2P и синхронизация (обязательно учитывать во всех задачах)

Программа работает **без облака и центрального сервера**.

Соединение между устройствами:
- **Основной вариант:** WebRTC (прямое Peer-to-Peer соединение).
- Для обхода NAT используется бесплатный **STUN-сервер** (передаёт только IP-адреса для handshake, данные не проходят через него).
- **Альтернативный и более надёжный вариант:** Mesh VPN (Tailscale / ZeroTier / Headscale). Устройства «видят» друг друга как в одной локальной Wi-Fi сети, даже если находятся в разных городах и за NAT.

Синхронизация данных:
- CRDT + event-sourcing.
- Фото хранятся локально (`app_data_dir/flower_photos/`), по сети передаётся только путь.
- Версионность: `app_version`, `db_schema_version`, `min_compatible_version`.
- Совместимость проверяется при каждом подключении.

**Важно для кода:** любые новые таблицы и команды должны быть совместимы с системой event-sourcing и не ломать CRDT.

## 4. Базы данных

- Для каждой предустановки создаётся **отдельная SQLite база**.
- Миграции только аддитивные (никогда не удаляем и не меняем существующие колонки).
- Общие таблицы для режима flowers (актуально на момент SPEC_v2):
  - `flower_sorts`, `greenhouse_raw`, `packaging_log`, `orders`, `order_items`, `greenhouse_harvest_log`, `app_settings` и др.
- Настройки хранятся в таблице `app_settings` (key-value).

## 5. Правила работы Claude с кодом (обязательные)

1. **Всегда** сверяйся с `SPEC_v2.md` — это актуальное техническое задание.
2. Все изменения в режиме «flowers» должны быть **изолированы** проверками `$preset === 'flowers'`.
3. Никогда не ломай совместимость версий.
4. UI/UX приоритеты:
   - Минимальное удивление, максимальная ясность.
   - Glass-morphism, hover/long-press на карточках.
   - i18n (английский + русский).
   - Валюта по умолчанию — грузинский лари (₾).
5. Фото и файлы: сохраняй только относительные пути, файлы — в `app_data_dir`.
6. Печать: используй `window.print()` + CSS @media print (никаких PDF-библиотек).
7. Никаких background tasks в Tauri (кроме того, что уже реализовано).
8. Все новые команды Rust должны быть в `src-tauri/src/commands/`.
9. Все типы TypeScript — в `src/lib/tauri/types.ts`.
10. Сторы Svelte — в `src/lib/stores/`.
